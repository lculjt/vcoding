# AI 内容生产与分发平台需求计划书

## 0. 文档定位

本文是 AI 内容生产与分发平台一期的唯一详细需求源。涉及一期页面、字段、数据表、接口、里程碑和验收标准时，以本文为准。

AI 产品套件的顶层路线图见 [AI 双系统产品规划](../ai-product-suite/dual-system-roadmap.md)。竞品与差异化分析见 [竞品分析](competitor-analysis.md)。

## 1. 项目背景

当前内容生产通常需要人工完成选题、撰稿、改写、视频脚本、平台格式适配、发布检查和发布记录维护。不同平台对标题、正文长度、标签、封面、视频脚本、审核规则和发布方式要求不同，导致内容团队需要重复加工同一份内容。

本项目计划建设一个 AI 内容生产与分发平台，帮助用户从一个选题出发，生成文章、短视频脚本和多平台发布包，并逐步接入具备官方开放能力的平台，实现自动或半自动发布。

## 2. 产品定位

平台定位为“AI 内容生产工作台 + 多平台分发管理系统”。

核心目标不是简单聊天生成文本，而是围绕内容从生产到分发形成闭环：

```text
选题
  ↓
AI 生成内容
  ↓
平台格式适配
  ↓
人工审核
  ↓
生成发布包
  ↓
自动发布或辅助人工发布
  ↓
记录发布结果
```

## 3. 目标用户

- 自媒体创作者
- 内容运营人员
- 企业市场团队
- 短视频运营团队
- 需要批量生成内容并分发到多平台的个人或团队

## 4. 一期目标

一期目标是完成“从选题到多平台发布包”的最小闭环。

一期不强承诺所有平台自动发布。对于已开放官方发布 API 且账号具备权限的平台，后续可接入自动发布；对于暂不具备开放 API 或权限不明确的平台，一期提供发布包、复制内容、下载素材和跳转发布后台等半自动能力。

一期核心产出：

- 一篇标准文章
- 多个平台版本的内容
- 一个短视频脚本
- 标题、摘要、标签、封面提示词
- 发布检查结果
- 发布任务记录

## 5. 功能范围

### 5.1 选题管理

用户可以创建和管理内容选题。

字段建议：

- 选题标题
- 内容方向
- 目标受众
- 核心关键词
- 内容类型：文章、公众号文、小红书文案、知乎回答、短视频脚本
- 语气风格：专业、轻松、营销、科普、测评、故事化
- 期望字数
- 状态：草稿、生成中、待审核、已完成、已归档

### 5.2 AI 内容生成

基于选题生成内容资产。

一期支持：

- 生成标题候选
- 生成文章大纲
- 生成正文
- 生成摘要
- 生成标签
- 生成短视频脚本
- 生成封面图提示词
- 支持重新生成
- 支持局部改写

### 5.3 多平台适配

同一份内容生成不同平台版本。

一期建议支持以下平台格式：

- 微信公众号版：结构完整、段落清晰、适合长文阅读
- 小红书版：短句、强钩子、标签明显、适合种草
- 知乎版：逻辑更强、论证更完整
- 抖音/视频号版：口播脚本、分镜、开头钩子
- B 站版：标题、简介、标签、视频脚本

### 5.4 内容审核

发布前进行基础检查。

一期检查项：

- 标题是否为空
- 正文是否为空
- 摘要是否为空
- 标签是否为空
- 平台字数是否超限
- 是否缺少封面提示词
- 是否存在敏感词
- 是否存在明显格式问题

审核状态：

- 待审核
- 审核通过
- 审核驳回

### 5.5 发布包管理

平台根据不同渠道生成发布包。

发布包内容：

- 平台名称
- 标题
- 正文或脚本
- 摘要
- 标签
- 封面提示词
- 素材清单
- 发布说明
- 复制按钮
- 下载 Markdown
- 下载 Word 或文本文件

### 5.6 发布任务

记录内容发布计划和执行结果。

字段建议：

- 内容 ID
- 平台
- 账号
- 发布方式：自动发布、半自动发布、人工发布
- 计划发布时间
- 实际发布时间
- 发布状态：待发布、发布中、发布成功、发布失败、人工处理
- 失败原因
- 平台返回 ID
- 平台链接

### 5.7 平台账号管理

一期先做账号登记，不强制接入 OAuth。

字段建议：

- 平台名称
- 账号名称
- 账号类型
- 发布能力：自动发布、半自动发布、仅生成发布包
- 授权状态
- 备注

## 6. 发布能力边界

平台发布能力需要按官方开放平台逐个确认。

一期建议策略：

| 平台 | 一期能力 | 后续方向 |
| --- | --- | --- |
| 微信公众号 | 生成文章发布包，后续可接入草稿和发布接口 | 接入官方草稿、发布、状态查询接口 |
| 抖音 | 生成视频脚本和发布包，具备条件后接入官方发布 API | 接入抖音开放平台内容发布能力 |
| B 站 | 生成视频标题、简介、标签和脚本 | 调研并接入视频稿件发布能力 |
| 小红书 | 生成笔记发布包，人工复制发布 | 确认开放权限后再接自动发布 |
| 知乎 | 生成回答或文章发布包 | 视官方开放能力决定是否自动发布 |

产品文案不得承诺“支持所有平台一键发布”。建议对外表述为：

```text
支持多平台内容适配和发布包生成；已接入官方开放能力的平台支持自动发布，未接入平台支持半自动发布。
```

## 8. 后端模块建议

后端模块建议命名：

```text
backend/vcoding-content
```

前端应用建议命名：

```text
frontend/apps/content-web
```

核心后端包结构：

```text
api/
application/
domain/

```

核心领域对象：

- Topic：选题
- ContentDraft：内容草稿
- PlatformContent：平台适配内容

## 9. 数据表初稿

一期建议表：

- `ct_topic`：选题表
- `ct_content_draft`：内容草稿表
- `ct_platform_content`：平台适配内容表
- `ct_review_record`：审核记录表
- `ct_publish_task`：发布任务表
- `ct_platform_account`：平台账号表

## 11. 一期里程碑

### 阶段 1：内容系统骨架

- 新增 `content-web`
- 新增 `vcoding-content`
- 接入统一登录鉴权
- 完成选题 CRUD

### 阶段 2：内容生成闭环

- 完成内容草稿模型
- 完成 AI 生成接口占位
- 完成标题、大纲、正文、脚本生成结果保存

### 阶段 3：多平台适配

- 完成平台内容版本模型
- 支持公众号、小红书、知乎、抖音/B站脚本格式适配
- 支持平台版本编辑

### 阶段 4：审核与发布包

- 完成审核流程
- 完成发布任务
- 完成复制、下载发布包
- 支持人工记录发布结果

## 12. 一期验收标准

一期完成后，应满足：

- 用户登录后可以进入内容系统。
- 用户可以创建选题。
- 用户可以基于选题生成文章内容。
- 用户可以生成至少 3 个平台版本。
- 用户可以审核平台版本。
- 用户可以生成发布包。
- 用户可以创建发布任务。
- 用户可以手动记录发布结果。
- 未登录访问内容系统时，跳转统一登录页。

## 13. 非一期范围

以下能力不放入一期：

- 所有平台自动发布
- 平台 OAuth 授权闭环
- 视频自动剪辑
- 自动生成真实封面图片
- 自动评论互动
- 数据看板和粉丝增长分析
- 多租户计费
- 团队协作审批流

## 14. 风险与约束

- 各平台发布 API 权限不同，自动发布需要逐个平台确认。
- 内容生成质量依赖模型、提示词和上下文。
- 发布前必须保留人工审核环节，避免违规内容直接发布。
- 自动发布能力要遵守各平台开放平台协议。
- 若后续接入平台账号授权，需要单独设计 OAuth、Token 存储、刷新和权限隔离。

后续开发顺序见 [开发路线](development-roadmap.md)。

当前下一步建议先实现内容系统骨架：

```text
content-web
vcoding-content

```

先把“登录后进入内容系统并管理选题”跑通，再接 AI 生成能力。
