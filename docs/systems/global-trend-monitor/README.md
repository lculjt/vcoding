# 海外热点聚合观察台文档

本目录用于维护海外热点聚合观察台的需求分析、数据源接入策略、阶段计划和后续设计说明。

## 文档列表

- [一期需求分析文档](phase-1-requirement-analysis.md)
- [一期任务文档](phase-1-task-plan.md)
- [一期技术设计文档](technical-design.md)
- [一期数据源验证记录](phase-1-source-validation.md)

当前实现进度：一期正式范围已收缩为 YouTube 和 GitHub。后端已完成两个数据源的采集器契约和最小实现，支持管理员手动采集、平台内幂等更新、指标快照、热度评分、任务统计和可配置每日定时任务。YouTube 正式采集仍需配置 `VCODING_YOUTUBE_API_KEY`。Hacker News 已移出一期运行范围，历史验证结论仅作为二期预研材料保留。

## 文档职责

`phase-1-requirement-analysis.md` 是一期需求分析源，维护：

- 项目背景和产品定位
- 目标用户和核心场景
- 海外热点数据源建议
- 一期功能范围和非一期范围
- 数据采集、热度计算、内容查看和归档需求
- 合规边界和平台接入风险
- 页面规划、数据对象、接口初稿和验收标准

`phase-1-task-plan.md` 是一期开发任务源，维护：

- 阶段拆分
- 前后端任务清单
- 数据源验证任务
- 图表和数据洞察任务
- 阶段验收标准
- 暂缓任务边界

`technical-design.md` 是一期实现设计源，维护：

- 前后端模块边界和目录结构
- 认证、权限和数据隔离方式
- 核心数据表和状态模型
- 数据源 connector 抽象和失败重试规则
- 去重、热度评分和日聚合口径
- 热点、图表、数据源和任务接口设计
- 图表交互、可观测性和合规存储边界

`phase-1-source-validation.md` 是数据源验证记录，维护：

- YouTube、GitHub 的一期官方接口和字段口径
- Hacker News 的二期预研验证记录
- 认证、限流和配置要求
- 最小验证接口和实测结果
- 正式 connector 开发前的结论和待办

后续进入开发时，可在本目录继续补充接口示例、数据源验证记录、部署说明和迭代复盘。
