# vcoding 平台视觉规范

## 目标

本文用于统一 vcoding 平台所有前端应用的基础视觉方向。后续实现 `auth-web`、`portal-web`、业务系统前端、后台管理页面或通用 UI 组件时，应优先遵守本文。

当前阶段采用“开发者工具风格”：整体专业、克制、可信，带有适度技术感；避免营销站式大渐变、大插画、大口号和过度装饰。

## 设计关键词

- 专业：界面信息清晰，操作路径直接，不堆叠无关装饰。
- 克制：低饱和背景、清晰边界、轻量阴影，避免强烈视觉噪音。
- 技术感：使用深浅灰、冷蓝、青绿色点缀，允许少量代码、网格、终端、状态灯等视觉隐喻。
- 平台感：登录、门户、业务系统、管理页面要像同一套产品，而不是多个模板拼接。
- 可扩展：视觉规则应能支撑后续多个业务系统，不为单个页面做孤立风格。

## 色彩

### 基础色

```text
--vc-bg-page: #0f172a;
--vc-bg-canvas: #111827;
--vc-bg-surface: #ffffff;
--vc-bg-subtle: #f8fafc;
--vc-bg-muted: #eef2f7;
```

说明：

- 登录页可以使用深色开发者工具氛围背景，但表单区域必须保持高可读性。
- 门户、后台和业务系统主体优先使用浅色工作台，不做大面积纯深色界面。
- 深色只作为品牌区、侧栏、登录背景或代码感装饰使用。

### 主色和点缀色

```text
--vc-primary: #2563eb;
--vc-primary-hover: #1d4ed8;
--vc-primary-soft: #dbeafe;
--vc-accent: #14b8a6;
--vc-accent-soft: #ccfbf1;
```

说明：

- 主色用于主按钮、当前导航、关键链接和聚焦状态。
- 青绿色点缀用于状态灯、连接成功、系统在线、开发者工具感细节。
- 不使用大面积紫色、粉色、橙棕色或单一蓝紫渐变作为平台主视觉。

### 文本和边框

```text
--vc-text-strong: #0f172a;
--vc-text-main: #1e293b;
--vc-text-muted: #64748b;
--vc-text-subtle: #94a3b8;
--vc-border: #dbe3ee;
--vc-border-strong: #cbd5e1;
```

说明：

- 正文必须保证对比度，不能用过浅灰色承载重要信息。
- 表单错误、辅助提示、说明文案应层级清楚，不抢主操作。

### 语义色

```text
--vc-success: #16a34a;
--vc-warning: #d97706;
--vc-danger: #dc2626;
--vc-info: #2563eb;
```

语义色只用于状态表达，不作为页面装饰主色。

## 字体与排版

默认字体：

```text
Inter, ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif
```

等宽字体：

```text
"JetBrains Mono", "SFMono-Regular", Consolas, "Liberation Mono", Menlo, monospace
```

字号建议：

```text
页面主标题: 28px / 36px / 600
页面二级标题: 20px / 28px / 600
卡片或面板标题: 16px / 24px / 600
正文: 14px / 22px / 400
表单标签: 13px / 20px / 500
辅助说明: 12px / 18px / 400
按钮文字: 14px / 20px / 500
```

规则：

- 不使用负字距。
- 不用超大营销标题撑满登录页。
- 表单、侧栏、表格、工具栏等工作界面保持紧凑但不拥挤。
- 中文文案优先短句，避免在按钮、标签和菜单中出现过长文本。

## 布局与空间

基础间距：

```text
4px, 8px, 12px, 16px, 20px, 24px, 32px, 40px, 48px
```

页面规则：

- 登录页聚焦身份认证，不做营销落地页。
- 门户页突出系统入口、当前用户、最近访问和平台状态。
- 后台管理页优先信息密度、扫描效率和稳定布局。
- 业务系统页遵循平台通用导航、头部、内容区和状态反馈规则。
- 页面区块不做层层嵌套卡片；卡片只用于独立信息项、表单容器、列表项或弹窗。

## 组件风格

### 圆角

```text
小控件: 6px
输入框/按钮: 6px
卡片/面板: 8px
弹窗: 8px
```

除非组件库已有约束，不使用大圆角胶囊风格作为默认。

### 按钮

- 主按钮使用 `--vc-primary`。
- 次按钮使用白底、边框和深色文字。
- 危险按钮只用于删除、禁用、强制退出等破坏性操作。
- 图标按钮优先使用明确图标，并提供 hover/focus 状态。
- 按钮高度建议：普通 `36px`，紧凑 `32px`，强调 `40px`。

### 表单

- 输入框高度建议 `36px` 或 `40px`。
- 标签靠近输入框，错误提示紧跟字段下方。
- 验证码、短信验证码等流程必须有明确状态：可发送、倒计时、发送中、已发送、失败。
- 登录表单不展示 Token、JWT、Cookie 等技术细节。

### 面板和卡片

- 面板优先使用轻边框，不依赖厚重阴影。
- 阴影只用于弹窗、下拉层、浮层等真正需要层级的元素。
- 避免多个卡片套卡片。
- 信息密集页面优先使用表格、列表、分组和工具栏，不把所有内容做成卡片墙。

### 图标

- 优先使用 `lucide` 或 Element Plus 图标体系中的一致风格。
- 图标线宽、尺寸和颜色应统一。
- 不用随机 emoji 作为正式功能图标。

## 登录页方向

`auth-web` 是唯一登录界面，应体现开发者工具风格，但不能影响登录效率。

建议结构：

```text
左侧品牌/平台信号区
  - vcoding 标识
  - 简洁平台说明
  - 少量终端、连接状态、系统网格等技术感元素

右侧认证表单区
  - 密码登录
  - 手机号验证码登录
  - 注册入口
  - 清晰错误提示
```

规则：

- 登录页首屏必须直接展示登录表单。
- 不增加营销页导航、价格、案例、客户评价等无关内容。
- 背景可以有低对比度技术纹理，但不能干扰表单阅读。
- 登录成功后按 `redirect` 跳转；无 `redirect` 时进入门户。

## 门户与业务系统方向

门户和业务系统采用浅色工作台风格：

- 顶部或侧边导航稳定，不频繁变化。
- 当前用户、系统入口、未读状态、最近访问等信息清晰可扫。
- 业务系统复用平台色系和组件规则，但允许在局部使用业务图标或模块色。
- 模块色不得替代平台主色，避免各系统视觉割裂。

## 前端落地位置

推荐沉淀方式：

```text
frontend/packages/ui
  - 平台通用组件
  - 平台 CSS Variables
  - Element Plus 二次封装

frontend/packages/shared
  - 视觉常量名称
  - 通用枚举和状态

frontend/apps/*/src/styles
  - 应用级样式入口
  - 页面局部样式
```

后续实现真实页面时，应优先将可复用的颜色变量、组件尺寸、按钮/输入框风格沉淀到 `frontend/packages/ui`，避免每个应用复制一份样式。

当前公共样式落地文件：

```text
frontend/packages/ui/src/styles/
├── tokens.css       # 颜色、字体、字号、间距、圆角、阴影和动画变量
├── base.css         # box-sizing、body、链接、表单控件等基础样式
├── element.css      # Element Plus 主题变量和基础组件覆盖
├── components.css   # 平台通用 class，例如 vc-auth-card、vc-tech-grid-bg
└── index.css        # 统一样式入口
```

应用侧应通过以下方式引入公共样式：

```ts
import 'element-plus/dist/index.css'
import '@vcoding/ui/styles'
```

应用自己的 `src/styles/main.scss` 只放页面局部样式，不重复定义全局字体、body 背景和基础色彩变量。

## AI 生成界面约束

使用 AI 生成或修改前端界面时，必须遵守：

- 先读取本文，再实现页面。
- 不生成营销站式登录页。
- 不使用一整页同色系渐变背景。
- 不使用过度装饰的玻璃拟态、霓虹光效、漂浮色块。
- 不把后台/平台型系统做成卡片堆叠展示页。
- 必须检查移动端和桌面端文字不溢出、不重叠。
- 真实实现应与设计稿、截图或本文视觉规则一致，不能只保证功能可用。
